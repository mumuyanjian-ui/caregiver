# 陪诊应用设计方案

## 1. 设计目标

基于现有资料，这个应用不应只是“预约陪诊”，而应成为一个面向患者和家属的 **医院流程导航工具 + 陪诊服务入口**。  
核心目标有三点：

1. 让用户快速找到目标医院、目标科室、目标流程。
2. 让用户在院内明确知道“下一步去哪里、做什么、带什么”。
3. 当用户需要帮助时，能一键下单陪诊，并让陪诊员按标准 SOP 执行。

## 2. 从文件中提炼出的关键信息

### 2.1 Excel 表格里的高价值内容

现有表格已经具备一个很强的产品底座，主要分成 4 类数据：

1. **医院基础库**
字段包括：
- 医院名称
- 所在区
- 详细地址
- 总机号码
- 最近地铁/出口
- 步行距离
- 就诊高峰
- 核心科室
- 陪诊建议

这部分已经覆盖广州多个区、约 50 家医院，可直接作为首版医院库。

2. **医院科室匹配库**
字段包括：
- 科室类型
- 适合医院名称
- 备注

这部分适合做“按症状/按科室找医院”的推荐入口，比如：
- 产科 / 妇科 / 生殖
- 儿科 / 儿童康复
- 老年病 / 慢病 / 陪护
- 肿瘤 / 化疗 / 术后
- 骨科 / 运动损伤 / 术后
- 神经内科 / 神经外科
- 呼吸科 / 肺部疾病
- 中医 / 康复 / 针灸
- 眼科
- 口腔 / 种植 / 正畸
- 精神心理 / 老年痴呆
- 传染病 / 肝病
- 体检 / 入职 / 全家体检

3. **单医院详细门诊流程库**
以“中山大学附属第一医院（总院）”为样板，已拆出：
- 挂号方式
- 门诊楼层分布
- 辅助检查布局
- 取药流程
- 缴费/医保结算方式

4. **陪诊服务 SOP 库**
已经有完整标准流程：
- 接单与前期准备
- 上门接送
- 到院挂号建档
- 诊室沟通
- 检查与缴费
- 取药与核对
- 报告打印与复诊预约
- 送返与回访
- 禁止行为与服务规范

### 2.2 可直接产品化的关键流程

从表格看，用户最关心的不是“医院百科”，而是以下高频任务：

1. 我去哪个医院最合适。
2. 这家医院怎么挂号。
3. 到院后先去哪一层。
4. 检查去哪里做。
5. 做完怎么缴费。
6. 西药/中药去哪里拿。
7. 报告在哪里打印。
8. 如果老人、孕妇、术后患者不方便，怎么快速叫陪诊。

## 3. 产品定位

产品名建议：**陪诊通** / **就医陪行** / **医院流程助手**

产品定位：

**一个帮助用户看懂医院流程、完成挂号检查取药，并在需要时预约陪诊服务的就医助手应用。**

## 4. 目标用户

### 4.1 C 端用户

- 第一次去陌生医院的人
- 老人及其家属
- 孕产妇家庭
- 儿科家长
- 肿瘤、骨科、神经科等流程复杂患者
- 需要术后复诊、长期慢病复查的人

### 4.2 B 端/服务端用户

- 陪诊员
- 调度人员
- 运营维护人员

## 5. App 核心功能结构

### 5.1 首页

首页不是做成泛资讯，而是做成“任务入口”：

- 搜医院 / 搜科室
- 今天去医院
- 挂号流程
- 检查流程
- 取药流程
- 找陪诊
- 热门医院快捷入口

首页推荐卡片可展示：
- 广州热门医院
- 就诊高峰提醒
- 儿科/产科/肿瘤等高需求科室入口

### 5.2 医院详情页

每家医院详情页建议拆成 6 个 tab：

1. **医院概况**
- 医院名称
- 地址
- 电话
- 最近地铁
- 步行距离
- 就诊高峰
- 陪诊建议

2. **挂号**
- 预约挂号方式
- 当日挂号方式
- 放号时间
- 自助机位置
- 人工窗口位置
- 建档入口

3. **看诊路线**
- 门诊楼栋
- 楼层分布
- 对应科室
- 分诊报到点

4. **检查**
- 抽血位置
- 心电图位置
- B 超位置
- CT/MRI 位置
- 内镜位置
- 是否需要提前预约

5. **取药**
- 西药房位置
- 中药房位置
- 急诊药房位置
- 取药步骤
- 核对提醒

6. **缴费与报告**
- 手机缴费
- 自助机缴费
- 人工收费位置
- 医保结算位置
- 报告打印位置

### 5.3 科室找医院

基于“医院科室及陪诊信息表”，做一个非常实用的推荐页：

- 选择科室类型
- 返回推荐医院列表
- 展示推荐理由
- 展示是否适合老人/孕妇/儿童/术后患者

例如：
- “产科 / 妇科 / 生殖”推荐广医三院、省妇幼、市妇儿
- “眼科”推荐中山眼科中心
- “传染病 / 肝病”推荐市八医院嘉禾院区

### 5.4 院内流程导航

这是应用最有差异化的功能。

用户点击“我现在要做什么”，进入任务模式：

- 我要挂号
- 我要报到
- 我要去检查
- 我要缴费
- 我要取药
- 我要打印报告

每个任务都显示：
- 去哪里
- 怎么做
- 需要什么证件
- 是否需要预约
- 预计等待时间提示

例如中山一院的“取药”页面可直接显示：
- 西药：2 楼东侧西药房
- 中药：2 楼西侧中药房
- 急诊药：1 楼急诊药房
- 操作：先缴费，再扫码/排队，取药时核对姓名、药名、剂量、用法、有效期

### 5.5 陪诊下单

用户下单时不该只填时间地点，而要把表格里的 SOP 结构化：

- 患者姓名
- 年龄
- 病种/主要诉求
- 医院
- 科室
- 预约情况
- 行动能力
- 是否需要轮椅
- 是否需要上门接送
- 是否需要代办建档/挂号协助
- 陪同人数
- 既往病史/过敏史

### 5.6 服务过程页

把陪诊 SOP 变成服务过程可视化：

- 已接单
- 已联系客户
- 已确认证件资料
- 已出发
- 已到院
- 已挂号/建档
- 已完成问诊
- 已完成检查
- 已取药
- 已整理报告
- 已送返
- 已回访

这样用户会更安心，也便于平台做服务管理。

## 6. 重点页面设计

### 6.1 首页信息架构

- 顶部搜索框：搜医院/科室/流程
- 快捷按钮：挂号、检查、取药、陪诊
- 热门医院卡片
- 高需求科室专区
- 当前就医提醒卡

### 6.2 医院详情卡片

建议采用“信息卡 + 步骤卡”的结构，而不是纯文本。

示例模块：
- 基础信息卡
- 今日高峰提醒卡
- 地铁到院卡
- 挂号方式卡
- 楼层导览卡
- 检查路线卡
- 取药路线卡
- 陪诊下单卡

### 6.3 流程步骤卡

每一步统一为：
- 步骤名
- 位置
- 操作动作
- 注意事项

示例：

**步骤 1：挂号**  
位置：1 楼大厅左侧 / 公众号 / 自助机  
操作：实名注册 → 选择科室 → 缴费 → 取号/报到  
注意：中山一院提前 7 天 8:00 放号，当日号 7:30-17:00

### 6.4 取药页

用户价值非常强，建议单独做一级入口。

页面应包含：
- 西药 / 中药 / 急诊药切换
- 药房位置
- 取药方式
- 当前高峰提醒
- 药品核对清单
- 一键呼叫陪诊员协助取药

## 7. 首版产品功能建议

### 7.1 MVP 必做

1. 医院库
- 先接入表格中已有的广州医院数据

2. 科室推荐
- 基于科室类型推荐医院

3. 单医院流程页
- 先把中山一院作为模板医院

4. 院内任务导航
- 挂号、检查、缴费、取药、打印报告五大任务

5. 陪诊下单
- 支持填写基础需求并提交

6. 陪诊 SOP 进度页
- 可视化服务状态

### 7.2 第二阶段

1. 扩展更多医院的详细流程模板
2. 增加楼层地图/路线图
3. 增加老人模式和大字模式
4. 增加复诊提醒与报告归档
5. 增加陪诊员评分与标签体系

## 9. 业务亮点设计

### 9.1 “流程比预约更重要”

市场上很多陪诊产品只强调“有人陪”，但你这份资料最大的优势是：

**已经掌握了医院流程知识库。**

所以产品竞争力应该是：

- 先解决“我怎么走流程”
- 再承接“我需要人帮忙”

### 9.2 从单医院模板复制

中山一院的数据已经足够标准，适合做模板：

- 挂号模块模板
- 楼层模块模板
- 检查模块模板
- 取药模块模板
- 结算模块模板

后续新增医院时，运营只需按模板录入即可。

### 9.3 用户最强感知功能

如果只能做 3 个最打动人的功能，建议优先做：

1. **医院流程导航**
2. **取药/检查快捷入口**
3. **一键陪诊下单**

## 11. 运营落地建议

### 11.1 内容建设优先级

先做深，不先做广：

1. 中山一院做成标杆医院
2. 再补省人民、省中医院、广医三院、珠江医院、中山肿瘤
3. 再逐步扩展广州全院

### 11.2 数据维护方式

建议后台支持以下编辑维度：

- 医院基础信息
- 科室推荐关系
- 流程步骤
- 楼层位置
- 取药位置
- 陪诊提示
- 最近更新时间

并且每条流程要支持“已核验/待核验”状态。

## 12. 结论

基于这两份文件，尤其是 Excel 表格，最适合做的不是泛健康 App，而是一个 **“医院流程导航 + 陪诊服务”双核心应用**。

最合理的首版产品路径是：

1. 先把“医院库 + 科室推荐 + 中山一院详细流程”做出来；
2. 再把“挂号/检查/取药/报告打印”做成任务式导航；
3. 最后接入“陪诊下单 + SOP 进度追踪”。

这样既能让普通用户直接使用，也能自然转化为陪诊服务订单。

Excel 内容已足够支撑本方案的主体设计。
